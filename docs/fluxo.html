Tools vs. sem tools — passo a passo

Veja, etapa por etapa, o que acontece quando você manda um prompt. 👤 é você; 🐙 é o Polypus nos bastidores. Escolha o modo e avance.

0 / 0

Lado a lado

📝 Emulado (sem tools)

  • Como as ações nascem: por um protocolo de tags XML no prompt → o modelo escreve texto e o Polypus parseia.
  • Resultado de volta: como texto de usuário (<polypus:tool_result>).
  • Formato: depende do modelo — com redes de segurança (re-prompt, parser tolerante).
  • Funciona com: qualquer modelo, inclusive locais/base.

🔧 Nativo (com tools)

  • Como as ações nascem: pelo campo tools da API → o modelo devolve uma tool_call estruturada.
  • Resultado de volta: por uma mensagem estruturada (role: tool).
  • Formato: garantido pela API.
  • Funciona com: modelos que têm function-calling.

Em ambos, o loop do agente é idêntico — só muda como as tool calls nascem e voltam.
